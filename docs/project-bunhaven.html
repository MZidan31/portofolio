Data Mining Apriori Algorithm

Strategi Optimasi Penjualan Bun Haven Bakery

Market basket analysis implementation via KNIME.

1. Situation

Toko roti Bun Haven menghadapi asimetri operasional terkait perputaran stok produk dan inefisiensi konversi kampanye promosi. Titik buta (*blind spot*) dalam memahami pola pembelian lintas-produk mengakibatkan kehilangan momentum transaksi (*missed upselling opportunities*).

2. Task & Role

Mengeksekusi siklus hidup analitik end-to-end: dari rekayasa arsitektur bisnis, pra-pemrosesan data transaksi historis (2016-2017), hingga desain model penambangan aturan asosiasi. Tujuannya adalah mendeteksi probabilitas kombinasi pembelian yang presisi.

3. Action

Alur kerja di KNIME Analytics Platform dirancang untuk mengagregasi data dari node File Reader. Transaksi waktu-nyata diisolasi menggunakan node Group By, kemudian ditransformasi dengan String to Collection sebelum dilempar ke mesin Association Rule Learner menggunakan algoritma Apriori.

4. Result & Business Impact

Pemodelan mengungkap korelasi absolut, misalnya matriks pembelian {Toast} -> {Coffee} menghasilkan metrik vital:

  • Support: 0.033
  • Confidence: 0.704 (70.4% pembeli Toast akan membeli Coffee)
  • Lift: 1.48 (Signifikansi kausalitas tinggi)

Kecerdasan data ini langsung diterjemahkan menjadi arsitektur bisnis taktis:

  • Tata Ruang: Penempatan produk berkorelasi kuat (Kopi & Roti Bakar) dalam radius fisik yang berdekatan.
  • Product Bundling: Peluncuran SKU "Toast & Coffee Special" yang mendongkrak visibilitas keranjang belanja.
